Markdown

O conteúdo das notas é escrito na linguagem Markdown. Esta é uma linguagem criada para transformar texto sem formatação em HTML. Para saber mais sobre isso visite esta página do criador da linguagem.

Temas

Papel de parede

No editor de temas você pode definir um papel de parede personalizado. É preciso fornecer um URL da imagem que você quer usar. Pode ser da Internet mesmo, mas tenha em mente que caso a imagem não esteja no seu computador pode haver problemas para exibi-la, como quando você estiver offline.

Se você digitar apenas o nome da imagem, será carregada essa imagem da pasta wallpapers. Caso não seja incluída uma extenção de arquivo válida, o padrão será de uma imagem JPEG.

Dica: para saber o URL de uma imagem no seu computador, basta arrastá-la para dentro do navegador. O URL estará lá em cima (começa sempre com file:///).

Posição do papel de parede

O campo posição do papel de parede precisa de dois valores separados por espaço. O primeiro é o deslocamento horizontal e o segundo é o deslocamento vertical. Pode ser em porcentagem, como por exemplo 50% 50% (centralizado), por palavras chave (top, bottom, left, right, center), top right alinha no canto superior direito, ou ainda em pixels (por exemplo 30px 80px).

Cores

Todas as cores devem ser escritas como valores de cores CSS.

Cópias de segurança

Todos os dados são salvos no armazenamento local do navegador. Ao limpar dados de sites você estará apagando também as informações salvas pela Projetiva. Para evitar perdas de dados é recomendável criar uma cópia de segurança pelo link no painel de opções antes de apagar o histórico de navegação.

Atualização

Infelizmente a atualização terá que ser manual. Você tem duas opções para atualizar:

  1. Descompactar a pasta em qualquer lugar, abrir o arquivo start.html e defini-lo como página inicial do navegador. Caso não apareçam suas informações, crie uma cópia de segurança na versão antiga e então use o link "restauração" na nova versão para importar essas informações. No Chrome não foi necessário a cópia e restauração, mas no Firefox sim.

  2. Excluir a pasta antiga e copiar a nova no mesmo local, renomea-la para que fique com o mesmo nome da antiga. Não é preciso fazer mais nada porque o caminho do arquivo permanecerá inalterado. Mesmo assim recomendo uma cópia de segurança antes do processo para evitar eventuais frustrações.
